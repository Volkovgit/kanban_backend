### Критические пробелы безопасности (должны быть устранены)

| # | Пробел | Влияние на безопасность | Серьёзность |
|---|--------|-------------------------|-------------|
| 1 | **Требования HTTPS/TLS** | Передача учётных данных открытым текстом | 🔴 Critical |
| 2 | **Хранение/отзыв токенов** | Нет выхода, украденные токены остаются валидными | 🔴 Critical |
| 3 | **Rate Limiting** | Уязвимость к brute force/DoS | 🔴 Critical |
| 4 | **Раскрытие информации** | Сообщения об ошибках раскрывают детали системы | 🟠 High |
| 5 | **Шифрование данных-at-rest** | Компрометация базы данных раскрывает все данные | 🟠 High |
| 6 | **Предотвращение XSS** | Пользовательский ввод может внедрять скрипты | 🟠 High |
| 7 | **Предотвращение CSRF** | Межсайтовая подделка запроса | 🟡 Medium |
| 8 | **Безопасные заголовки** | Отсутствуют заголовки безопасности транспорта | 🟡 Medium |

### Рекомендации

**Перед `/speckit.plan`:**

1. **Добавить требования безопасности транспорта**
   - Принудительный HTTPS
   - Требование TLS 1.2+
   - Заголовки HSTS

2. **Указать lifecycle токена**
   - Отзыв токена при выходе
   - Механизм refresh токена
   - Обработка конкурентных сессий

3. **Добавить rate limiting**
   - Лимиты на IP и пользователя
   - Пороги для конкретных endpoint'ов
   - Митигация DDoS

4. **Определить стандарты сообщений об ошибках**
   - Общие сообщения об ошибках
   - Никаких stack traces в production
   - Предотвращение перечисления пользователей

5. **Добавить требования защиты данных**
   - Шифрование данных-at-rest
   - Безопасное управление ключами
   - Безопасность резервных копий

6. **Указать безопасные заголовки**
   - Content-Security-Policy
   - X-Frame-Options
   - HSTS

### Заметка об отслеживаемости

Все элементы ссылаются на разделы спецификации где применимо. Элементы помеченные `[Gap]` указывают на отсутствующие требования безопасности которые должны быть добавлены перед реализацией. Пробелы безопасности помеченные 🔴 Critical должны быть устранены как высший приоритет.

### Референс лучших практик безопасности

Этот чеклист соответствует:
- OWASP API Security Top 10
- OWASP Application Security Verification Standard
- NIST Cybersecurity Framework
- GDPR Articles 32, 33, 34 (security of processing)
